262  * \brief Put the system to idle or sleep or blocks in a WFI() waiting for the next tick, if neither
263  * idle nor sleep is possible.
264  *
265  * \detail Puts the system to idle or sleep, if possible. If an exit-from-idle or a wake-up is
266  * needed, it programs the Timer1 to generate an interrupt after the specified idle or sleep
267  * period. Else, the system stays forever in idle or sleep mode.
268  * If neither idle nor sleep is possible, it blocks in a WFI() call waiting for the next
269  * (already programmed) OS tick to hit.
270  *
271  * \param[in] xLowPowerPeriods The number of (prescaled) low power clock periods the OS will be
272  * idle. If it is 0 then the OS indicates that it can block forever waiting for an
273  * external event. If the system goes to sleep, then it can wake up only from an external
274  * event in this case.

276  * \return void
